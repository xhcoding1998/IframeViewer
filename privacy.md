《Iframe 检测器》隐私权政策

最近更新日期：2026-02-26
生效日期：2026-02-26

感谢您使用「Iframe 检测器」Chrome 扩展（以下简称“本扩展”）。我们非常重视您的隐私与数据安全。本隐私权政策旨在向您清晰说明本扩展在使用过程中如何处理（实际上是不收集）您的任何数据。

一、我们收集哪些信息

本扩展在设计上秉持“最小权限、零收集”的原则：

1. 本扩展不会收集、存储或上传您的任何个人身份信息（如姓名、邮箱、账号信息等）。
2. 本扩展不会将您访问的网站 URL、页面内容、截图或其他浏览记录上传到任何服务器。
3. 本扩展不会通过第三方分析工具（如 Google Analytics 等）对您的使用行为进行统计或跟踪。

所有操作（包括扫描页面中的 iframe、读取 iframe 的 src、解析 URL 参数、对当前标签页进行截图并裁剪等）均在您的本地浏览器中完成。

二、权限说明及用途

根据 Chrome 扩展机制，本扩展在 manifest 中声明了以下权限，仅用于实现核心功能：

1. activeTab / tabs
   - 作用：获取当前活动标签页的基础信息（如 tabId、windowId、URL），用于：
     - 在当前页面扫描并识别所有 iframe；
     - 在您点击「打开 src」「在新标签页中打开」等操作时，跳转到相应链接；
     - 调用截图接口时，确定是对哪一个标签页进行截图。
   - 本扩展不会将这些信息上传到任何远程服务器。

2. scripting
   - 作用：将用于扫描 iframe、滚动并定位 iframe 位置、更新 iframe src 等逻辑的脚本注入到当前页面，以便实现：
     - 获取页面上所有 iframe 的位置、尺寸、src 等信息；
     - 在您编辑参数后，将新的 URL 应用回对应的 iframe；
     - 在截图前将目标 iframe 滚动到可见区域，以获得更准确的快照。
   - 脚本仅在您当前浏览器中运行，不会把处理结果发送到外部服务器。

3. storage（当前版本中未实际使用）
   - 作用（预留）：未来可能用于本地保存一些纯配置类的偏好设置，例如界面选项、默认行为等。
   - 即便使用，也仅存储在浏览器本地的 chrome.storage 中，不会上传到开发者或第三方服务器。

4. <all_urls>（host_permissions）
   - 作用：允许本扩展在您打开的各个网站中读取 iframe 元素，用于：
     - 枚举页面上的所有 iframe；
     - 获取 iframe 的 src、尺寸、是否在视口内等技术性信息。
   - 本扩展不会利用该权限读取或上传与业务无关的页面内容，不会监控您的浏览行为。

三、数据的处理方式

1. 本地处理
   - iframe 列表、URL 参数解析结果、截图裁剪等全部逻辑在您的浏览器本地内存中进行。
   - 截图数据（data URL）只在本机内存 / popup 页面中短暂存在，用于展示和（如果您主动点击）下载图片文件。

2. 不上传、不共享
   - 本扩展不会将任何数据上传到开发者服务器或第三方服务。
   - 本扩展不会与任何第三方共享您的数据。

3. 截图与文件下载
   - 当您主动点击「快照」时，本扩展调用 Chrome 提供的 `chrome.tabs.captureVisibleTab` 接口，对当前可见页面进行截图，并在本地通过 Canvas 裁剪得到 iframe 区域的图像。
   - 当您点击「保存截图」时，生成的图片文件直接由浏览器下载到您的本地设备，开发者无法访问该文件。

四、第三方网站与内容

本扩展的核心用途是帮助您查看和分析网页中的 iframe 内容和 URL 参数。因此，在使用本扩展时，您仍然在访问各个具体网站，这些网站可能有各自独立的隐私政策。

请您知悉：

1. 本扩展不对任何第三方网站的隐私实践负责。
2. 当您通过本扩展打开或预览某个 URL 时，您与该网站之间的交互行为仍然受该网站自身的隐私政策与使用条款约束。
3. 我们建议您在使用任何网站或在线服务前，先阅读并了解其隐私政策。

五、儿童隐私

本扩展面向一般用户，不针对儿童设计或推广。本扩展不主动收集任何用户的个人信息，因此也不存在针对儿童的数据收集行为。

六、政策更新

随着功能的迭代或相关法律法规的变化，我们可能会适时更新本隐私权政策。若未来版本引入了任何形式的数据收集或上传行为，我们将：

1. 在隐私权政策中做出清晰说明；
2. 尽可能在扩展详情页中标注变更要点；
3. 在合理范围内征得您的知情和同意。

在该等更新生效后，您继续使用本扩展，即视为您接受更新后的隐私权政策。
